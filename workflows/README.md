# Workflows

Markdown SOPs that define objectives, required inputs, tool sequences, and edge cases.

## Workflow Template
Each workflow should contain:
1. **Objective** — what this workflow achieves
2. **Inputs** — what is required to start
3. **Steps** — ordered list of tool calls with expected outputs
4. **Edge Cases** — known failure modes and how to handle them
5. **Outputs** — what success looks like
